Kick Start you first Front-end development project

Himanshu Saraswat | Saturday,October 28,2017

Want to start your journey as a front-end developer/UI engineer?
Struggling to set-up the perfect development environment. Wondering where to store the code and what could be the most economical or rather zero cost way to launch your first project. If all these questions bewilder you then here is how you should have a rock solid start. Lets begin with answering a simple question first.

What should be the key asks for our first Front-End project ?

  • Text Editor - To pen down our design implementations
  • An angel browser which supports debugging
  • A file synchronization and interaction tool to speed up our development
  • Code repository
  • A platform to host and showcase your work

Which text editor should I use?
Some key features which I feel should help us in the quest for our perfect text editor are :

  • Code suggestions & autocompletion
  • Repository connectivity
  • Should look pleasing to my teary coding eyes

We all are familiar with the much respected Notepad++, will do the necessary job , but we need something a bit more advanced and way more interesting

Here comes Atom-The hackable editor.

Provides myriad of plugins to choose from, which will cater to all our code formatting, syntax correction and code language detection needs.

The interface is pretty neat and soothing ,but what I like the best is, its connectivity to the GitHub repository. Commiting and pushing the code to your personal repository is a piece of cake.

Don't worry, we shall be covering the magics of GitHub right below soon.

Getting your perfect text editor is a very personal choice , but, as we move forward in our development journey we need to get things done in the least possible time and in the best possible manner.
Moreover, for all my tech-boffins out there, Atom editor is hackable to the core.

And yes, it's free.

Very useful & comprehensive documentation available @Atom Docs

While coding it's very cumbersome to go and refresh your browser repeatedly to see the HTML/CSS changes, though I don't like someone keeping a watch over me when am coding, but I really want someone to watch over my files for any changes and synchronize them by itself but, how ?

The answer is Browsersync

What is Browsersync?

It's a file synchronization & interaction tools , which is a real helping hand, reducing your development time considerably.

Live reloading & URL pushing are the prime features that we will be using & it's all free.
It hardly takes 5 minutes to get browsersync up & running.

Install an appropriate Node.JS version for your operating platform.

Node.js

Follow the usual install procedure, to check if node got installed properly, open your terminal window and type: 

node -v

Always try to close the terminal after installation. Enter the command on re-opening the terminal, should reflect the current version that got installed.

To install browsersync, what we actually need is the node package manager, to check if that got installed correctly enter the below command in your terminal:

npm -v

You should be able to see the current version installed.

With all this accomplished, enter the below command, to install browsersync globally.

npm install -g browser-sync

By now you must have figured out what am going to say next, "to check if browsersync got installed correctly", yes exactly, you know what to do.
Browsersync official documentation@Get started in 5 minutes.

How does browsersync help me?

With all installations now in place, let's start our amazing development set-up, to run browsersync, go to the directory were you have your code stacked, open the terminal window & enter:

browser-sync start --server --directory --files "**/*"

This above command shall start a server on port 3000, watching all the files in the current directory, tadaaaaa!, this is exactly what we wanted right

Now, when you hit Ctrl+S in your editor, browsersync will automatically sync your changes and auto re-load the URL for you with .

Now, that we have an amazing editor, an angel renderer a.k.a browser & a time saver synchronizer, you can definitely code better & faster.

Did I hear showcase, yes, you need to bring all your efforts & thoughts out to the world, to know that you have arrived, for all that little leverage in your résumé, for all that inquisitive hobbyist in you, you need to get your work a platform.

Where should I showcase my project?

GitHub is an amazing repository, where you can not only store your code, but you can even collaborate with other developers, get your project code reviewed by your mentors and even showcase ,yes we are coming to the showcase part slowly.

When I created my first web project and wanted to share, the only thing that came to my mind was Google Drive, but people where not able see my code in working state and some or the other issue use to crawl in, that is when I decided to witch to GitHub.

How do you store code on GitHub ?

Signup on GitHub and create your first repository.

Install GIT on your machine selecting the apprpriate version


GIT

Follow the standard installation steps.
Now it's time to get your project some special space of its own. Clone the repository to your machine, yes it'll be empty at this moment.

Create a folder on your workstation "first amazing web-project", open the terminal window and enter:

git clone "your repository name"

You will see some sub-folders getting updated to your project folder.
Copy & paste your project code to this folder &"do not delete the .git folder which just got created"

Follow below steps to get your code to the online repository:
Open the termial window in your current project directory, yes the one which has the ".git" folder and enter: 

git add .

This commands will add all the folder in your project directory to your local repository & will get them ready for your very first commit.

Run the below command to check the status of your files and folders:

git status

All your files should appear in green indicating they they are good to be added to the Git repository

Make your first commit :

git commit -m "your commit message"

To push the code to the remote repository use the following:

git push

You might be asked for authorization credentials at this step, enter the credentials for your gitHub account & that's it, your code is live for millions to collaborate, review & share.
