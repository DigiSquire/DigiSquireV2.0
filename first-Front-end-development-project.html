Kick Start you first Front-end development project

Himanshu Saraswat | Saturday,October 28,2017

Want to start your journey as a front-end developer/UI engineer?
Struggling to set-up the perfect development environment. Wondering where to store the code and what could be the most economical or rather zero cost way to launch your first project. If all these questions bewilder you then here is how you should have a rock solid start. Lets begin with answering a simple question first.

What should be the key asks for our first Front-End project ?

  • Text Editor - To pen down our design implementations
  • An angel browser which supports debugging
  • A file synchronization and interaction tool to speed up our development
  • Code repository
  • A platform to host and showcase your work

Which text editor should I use?
Some key features which I feel should help us in the quest for our perfect text editor are :

  • Code suggestions & autocompletion
  • Repository connectivity
  • Should look pleasing to my teary coding eyes

We all are familiar with the much respected Notepad++, will do the necessary job , but we need something a bit more advanced and way more interesting

Here comes Atom-The hackable editor.

Provides myriad of plugins to choose from, which will cater to all our code formatting, syntax correction and code language detection needs.

The interface is pretty neat and soothing ,but what I like the best is, its connectivity to the GitHub repository. Commiting and pushing the code to your personal repository is a piece of cake.

Don't worry, we shall be covering the magics of GitHub right below soon.

Getting your perfect text editor is a very personal choice , but, as we move forward in our development journey we need to get things done in the least possible time and in the best possible manner.
Moreover, for all my tech-boffins out there, Atom editor is hackable to the core.

And yes, it's free.

Very useful & comprehensive documentation available @Atom Docs

ATOM

Now that we have our playground to code, where should be viewing what we develop ?

Which browser should I use to render & debug my HTML/Front-End code ?

From a developer friendly point of view what should be our assesing factors ?

  • Debugging support
  • Should be able to point out my code coverage
  • Should audit and assess my webpage for standard practices
  • Should be able to assist me in improving my page load time

And the winner is


Google Chrome- Not only hits every point in our check list but also gives us numerous plugins to integrate with all google services, for instance Google Analytics-which is really what we'll need to assess our work on the web.

All the chrome DevTools reaaly ease our life and help us create a masterpiece.

Below are some of the key tasks which will help you sharpen your development skills from the very beginning.

Want to know more on Chrome DevTools read the documentation @Chrome DevTools Overview

While coding it's very cumbersome to go and refresh your browser repeatedly to see the HTML/CSS changes, though I don't like someone keeping a watch over me when am coding, but I really want someone to watch over my files for any changes and synchronize them by itself but how ?

The answer is Browsersync

What is Browsersync?

It's a file synchronization & interaction tools , which is a real helping hand, reduces your development time considerably.

Live reloading & URL pushing are the prime features that we will be using and lastly its all free.

It takes hardly 5 minutes to get browsersync up & running.

Firstly install the Node.js version for your operating platform.

Node.js


Follow the usual install procedure, to check if node got installed properly, open your terminal window and type node --v

Should reflect the curent version that got installed

For browsersync what we need is the node package manager, to check if that got installed properly type npm --v in terminal. You should be able to see the current version installed.

With all this accomplished enter the below command to get browsersync.

npm install -g browser-sync
